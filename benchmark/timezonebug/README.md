# Timezone bug benchmark

这是一个用于测试 Hermes Agent 新模型适配性的本地代码修复题，不需要浏览器或联网搜索。

## 业务背景

中心节点收到的事件时间统一是 UTC 时间戳，但业务统计口径需要按照固定业务时区 `Asia/Shanghai` 的自然日聚合。

例如：

- `2026-05-16T16:30:00Z` 在上海时间是 `2026-05-17 00:30:00`
- 因此它应该计入业务日 `2026-05-17`，而不是 UTC 日期 `2026-05-16`

## 给模型的任务

请修复当前包中的失败测试：

1. 先阅读项目结构和相关代码，不要直接猜原因。
2. 运行现有测试，确认失败现象。
3. 定位失败测试背后的真实根因。
4. 做最小范围修复，优先修业务代码，不要为了让测试通过而硬改测试。
5. 补充至少 1 个能覆盖该 bug 的测试用例。
6. 再次运行相关测试，确认通过。
7. 最后给出简短报告：失败现象、根因、改动文件、为什么是最小安全修复、执行过哪些验证命令及结果。
